### `resize()` 參數寫法
```js
// ① 指定寬高
.resize(800, 600)

// ② 只指定寬（最常用）
.resize({ width: 800 })

// ③ 指定寬高並加其他選項（工作上常見）
.resize({
  width: 800,
  height: 600,
  fit: "cover",
})
```

---

### `resize()` 只設定寬度 `width` 或高度 `height` 時

`resize()` 可以只提供寬度或高度；只提供其中一項時，另一項會依圖片比例計算。當寬高同時提供時，才需要透過 fit 決定圖片如何配合指定框架。

由於圖片尺寸只能使用整數像素，計算結果若有小數，Sharp 會取整數，因此縮放前後計算出的長寬比可能有極小差異，但圖片並未實際變形。


可參考 task04.js `resize({ width:1000 })` 時的 console：

```text
原始檔案大小：262179 Bytes
原始檔案寬度：1920 px
原始檔案高度：1280 px
原始長寬比：1.5
==============================
縮放後檔案大小：53678 Bytes
縮放後寬度：1000 px
縮放後高度：667 px
縮放後長寬比：1.4992503748125936
==============================
縮放前後長寬比視為相同
微小差異來自高度像素必須取整數
```

### 使用 `Math.abs()` 判斷比例偏差值

`Math.abs()` 功能：回傳一個數字的絕對值（absolute value）。

測試 `resize({ width:1000 })` 圖片縮放前後長寬比時，使用 `Math.abs()` 來判斷比例差距大小。

```js
Math.abs(originalRatio - outputRatio);
```

之所以使用 `Math.abs()` ，是因為測試重點是在於差距大小，不看正負方向。

---

### `resize()` 的本質

resize 的本質不是「把圖片變小」，而是「改變圖片的尺寸（Dimensions）」。

它可以：

- 縮小（最常見）✅
- 放大（也可以）
- 改變長寬比例（如果設定方式允許）

以 task04 的測試結果來說：

```text
原始檔案大小：262179 Bytes
原始檔案寬度：1920 px
原始檔案高度：1280 px
原始長寬比：1.5
==============================
縮放後檔案大小：53678 Bytes
縮放後寬度：1000 px
縮放後高度：667 px
縮放後長寬比：1.4992503748125936
==============================
縮放前後長寬比視為相同
微小差異來自高度像素必須取整數
```

`resize({ width:1000 })` 把圖片從 1920×1280 縮放成 1000×667，總畫素數減少，因此圖片保留的細節會變少，圖片本身的像素尺寸也變小。如果之後再放大觀看，就會比原圖更不清楚；但在正常顯示尺寸下，通常不會明顯感覺到模糊。

所以 `resize()` 的本質不是單純「縮小」，而是重新決定圖片的尺寸與畫素數。只是實務上，絕大多數情況都是把大圖縮小，因此大家才常說它是「縮圖」。

---

### `resize()` 和 `quality` 的差異與實務搭配

- `resize()`：改變圖片尺寸與總像素數量，例如從 1920 × 1280 縮成 1000 × 667。
- `quality`：控制 JPEG、WebP 等有損格式的輸出壓縮品質，不會直接改變輸出圖片的寬高。

實務上常將兩者搭配：

```js
sharp(inputPath)
  .resize({
    width: 1000,
    withoutEnlargement: true,
  })
  .webp({
    quality: 80,
  })
  .toFile(outputPath);
```

### `fit` 的用途

**用途：決定圖片如何符合你指定的尺寸。**

注意：`fit` 主要在同時指定 `width` 和 `height` 時才有明顯作用。若只指定 `width`，Sharp 只需要依原始比例計算高度，通常不需要透過 `fit` 決定裁切或留白方式。

例如指定：

```js
.resize({
  width: 1000,
  height: 1000,
})
```

但是原圖是：

```text
1920 × 1280
```

比例是：

```text
3 : 2
```

而你要求：

```text
1000 × 1000
```

比例變成：

```text
1 : 1
```

Sharp 必須決定怎麼處理，這就是 `fit` 的工作。

以下為 `fit` 的各種設定值：

#### 1. `cover`（預設）

```js
.resize({
  width: 1000,
  height: 1000,
  fit: "cover",
})
```

效果：

```text
┌────────┐
│        │
│原始圖片 │
│        │
└────────┘
↓
裁掉左右或上下
↓
┌────────────┐
│            │
│   正方形    │
│            │
└────────────┘
```

原圖比例與目標比例不同時，會先等比例縮放到整個目標區域都被覆蓋，再裁掉超出的部分。

很適合：

* 大頭貼
* 商品縮圖

---

#### 2. `contain`


效果：

```text
┌────────────┐
│            │
│ 原始圖片    │
│            │
└────────────┘
↓
完整保留
↓
┌────────┐
│████████│
│圖片內容│
│████████│
└────────┘
```

會在剩餘空間補上背景；背景可能是透明、白色或其他指定顏色，取決於輸出格式與 `background` 設定。

例如原圖為 `1920 × 1280`，指定 `1000 × 1000` 時，圖片內容約為 `1000 × 667`，但包含背景後的最終輸出檔案仍是 `1000 × 1000`。

適合：

* Logo
* 海報
* 不想裁切圖片

---

#### 3. `fill`


效果：

```text
原本
😺😺😺😺
↓
😺😺😺
😺😺😺
😺😺😺
```

直接拉伸。

比例可能跑掉，通常比較少用。

---

#### 4. `inside`


作用：保持圖片比例，不裁切、不補背景，將圖片縮放到完全放得進指定尺寸的框框內，因此輸出的寬高可能小於指定值。

例如：

```text
最大允許

1000 × 1000
```

圖片可能變成：

```text
1000 × 667
```

不會裁切。

適合：

* 網頁圖片
* API 上傳限制
---

#### 5. `outside`

保持原始比例，將圖片調整為能完整包住指定寬高範圍的最小尺寸。輸出寬度與高度都會大於或等於指定值，但不會像 cover 一樣自動裁切成指定尺寸。

例如：
```
原圖：1920 × 1280
指定：1000 × 1000
fit: "outside"
```
輸出：約 `1500 × 1000`

因為它需要讓：
```
寬度 ≥ 1000
高度 ≥ 1000
```
又要保持 3:2 比例，所以會得到 `1500 × 1000`，且不裁切。

---          

### `withoutEnlargement` 的用途


**用途：不要把原本比較小的圖片放大。**

例如原圖尺寸為 `600 × 400`

若寫：

```js
.resize({
  width: 1000
})
```

如果沒有 `withoutEnlargement: true`，Sharp 會按比例放大為 `1000 × 667`，但畫質通常變差。因為 Sharp 雖然可以產生更多像素，但那些新像素不是相機原本拍到的真實細節，而是透過演算法根據鄰近像素推算出來的。

所以放大後可能出現：
```
- 邊緣變得較模糊
- 紋理與細節不足
- 某些圖像出現鋸齒或放大痕跡
- 像素數與檔案大小增加，但沒有新增原圖不存在的真實細節
```
等問題。


如果加上：

```js
.resize({
  width: 1000,
  withoutEnlargement: true,
})
```
Sharp 會發現原圖寬度只有 600px，因此維持原本的 `600 × 400`，不進行放大。

#### 實務常用例子

假設網站要求「所有圖片最大寬度 1000px。」

使用者可能上傳：

```
A：5000px
B：3000px
C：800px
```

希望得到：

```
A → 1000px
B → 1000px
C → 保持 800px（不要放大）
```

這就是 `withoutEnlargement: true` 最常見的用途。

--- 

### `resize()` 沒有 `maxWidth` 這個參數選項

Sharp 的 `resize()` 常用選項包括 `width`、`height`、`fit` 與 `withoutEnlargement`，沒有名為 `maxWidth` 的選項！

若想要設置最大寬度，應使用 `width` 搭配 `withoutEnlargement: true`。

```
產品參數 maxWidth
↓
轉換成 Sharp 的 width
↓
搭配 withoutEnlargement
```

### maxWidth 的設計可以解決什麼問題？

maxWidth 的核心不是「所有圖片都要變成相同寬度」，而是：**圖片寬度最多只能到某個值，超過才縮小，未超過就維持原尺寸。**

在 Sharp 中，這個需求不是由單一的 `maxWidth` 選項完成，而是由兩個設定共同實作：

- `width: maxWidth`：將目標寬度設為最大允許寬度。
- `withoutEnlargement: true`：當原圖寬度小於目標寬度時，不進行放大。

例如設定：
```js
const maxWidth = 1200;

sharp(inputPath).resize({
  width: maxWidth,
  withoutEnlargement: true,
});
```

使用者上傳：
```
4000px → 1200px
800px  → 800px
```
它可以解決幾個實際問題：

- **避免圖片尺寸遠超過顯示需求**

    假設網站內容區最多只會以約 1000px 寬顯示圖片，使用者卻上傳寬度 6000px 的圖片。瀏覽器仍可能需要下載完整的大圖，但畫面通常無法充分利用這麼高的像素尺寸，因此會造成不必要的傳輸與儲存負擔。

- **減少檔案大小與傳輸時間**

    圖片尺寸降低後，總像素數通常也會明顯下降，因此輸出檔案通常更小，網頁載入更快。

- **降低伺服器處理與儲存負擔**

    過大的圖片會消耗更多：
    ```
    記憶體
    CPU
    硬碟空間
    網路流量
    ```
    設定最大寬度後，輸出圖片的尺寸與檔案大小會較容易控制，避免使用者上傳極大的圖片，造成處理時間、記憶體使用量、儲存空間與網路流量突然增加。

- **避免小圖被無意義放大**

    搭配：
    ```
    withoutEnlargement: true
    ```
    小圖就不會被硬撐大。
